Breadcrumbs

What is it for

Displaying a horizontal list of labels that reflect the location of the current page in the hierarchy of a website provides context; helps the user understand where on the website they are and enables them to easily navigate up in the site hierarchy.

The labels should match the page titles and should function as hyperlinks to that page, with the exception of the last label which should be the title of the current page and should not link to itself.

Best Practices

  • Never display a breadcrumb on the site's topmost page.
  • Dynamically include the titles of user-generated content pages as appropriate.
  • Hyperlink all labels in the breadcrumb except the last one (which corresponds to the title of the current page.)
  • Do not use alone as the main navigation of the website.
  • Use when the page in question is placed fairly deep into the hierarchy of pages and when no other form of visual navigation can show the details of the same deep level.

Example

  • Level 1
  • Level 2
  • Level 3
  • Current page

Markup

<ul class="breadcrumbs">
        <li><a href="#">Level 1</a></li>
        <li><a href="#">Level 2</a></li>
        <li><a href="#">Level 3</a></li>
        <li class="active">Current page</li>
</ul>

Not-Forms Definition lists for labels and values outside of a form

What is it

Definition lists are the semantic way of showing a connection between a label and its value when there is no form structure to use.

Best Practices

  • Use when form elements are not available
  • There must be a meaningful link between the defintion title <dt> and the definition description <dd>
  • <dt> cannot support block level elements like headings
  • Do not replace complex tabular data
  • Does not contain screen reader accessibility features

Example

Name:
Elsa Bartley
Job Title:
User Interface and User Experience Designer
Company:
Microlise

Markup

<dl class="datalist">
      <dt>Name:</dt>
      <dd>Elsa Bartley</dd>   
      <dt>Job Title:</dt>
      <dd>User Interface and User Experience Designer</dd>
      <dt>Company:</dt>
      <dd>Microlise</dd>
 </dl>

Button elements

Buttons for actions

As a convention, buttons should only be used for actions while hyperlinks are to be used for objects. For instance, "Download" should be a button while "recent activity" should be a link.

Button styles can be applied to anything with the .btn class applied. However, typically you'll want to apply these to only <a> and <button> elements.

Multiple sizes

Fancy larger or smaller buttons? Add .btn-large pr .btn-small for two additional sizes.

Disabled state

For disabled buttons, add the .disabled class to links and the disabled attribute for <button> and input elements.

Primary Link Link

One class, multiple tags

The .btn class can be used on <a>, <button>, or <input> elements as required.

As a best practice, try to match the element for you context, if you have an input, use an <input type="submit"> for your button.
