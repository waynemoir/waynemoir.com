Accordion

What it is

An accordion is a grouped set of collapsible panels that provides access to a large number of links or other selectable items in a constrained space.

Each inlaid panel may be individually expanded (leaving the rest collapsed) by hovering on or clicking the title of (or an expand/collapse element on) the specific panel, to display a single subset of the options.

What Problem Does This Solve?

When there are too many items to fit into a limited space or when the number of items, if displayed all at once, would overwhelm the user, then the question is how to give the user access to all of the items in digestible chunks and without requiring scrolling, which can remove the user from the context or page position they may prefer.

When to use it

Use when the number of options is large, the space is constrained, and the list of items can be logically grouped into smaller, roughly equal sized chunks.

</div>

Best practices

  • Have the most important panel open by default, both to expose the important choices and to demonstrate the fact that each collapsed list can be individually opened.
  • Avoid nesting accordions inside of other accordions.

Date Picker

What it is

Date pickers provide a familiar calendar metaphor that resonates with most people and it helps them to think about dates in terms of where they are in the month and week. They help structure the date input in a way that your system can interpret reliably, thereby reducing validation errors caused by unrecognized date formats.

When to use it

Use this pattern when the person interacting with the application or experience would want to easily apply a date or date range to an item. This includes contexts where the application itself needs to apply date metadata ("date-stamp") a piece of information in order to track, filter, and sort items by by date or date order.

Common scenarios include events, tasks, and appointments.

Example use

Markup

<div id="ui-datepicker"></div>

Module Tabs

What it is

Tabbed navigation builds on the physical metaphor of a folders in a filing cabinet and is in tune with the typical users mental model.

Tabs provide a clear visual indication of what content can be found on a website and places the current location in context.

When to use it

When the user needs to navigate through one or more stacked panes of local content without refreshing the page.

Best practices

  • Never stack multiple rows of tabs to control a single piece of content.
  • Always show one tab as selected with its content visible.
  • The content should be visually connected to the tab it is associated with and be visually bounded (usually by a box).
  • Only one content area may be visible at a time.
  • Tabs should appear in the same location as the user clicks from one tab to another.
  • Selecting a tab will not affect other parts of the page.
  • Selecting a tab will not navigate to a different page or perform an action (beyond switching the visible content.

Slider

What it is

A control which enables the user to choose one distinct value within a finite range by dragging one or more sliders along a single axis.

When to use it

The user’s mental model of a series of values is closely tied to an increasing or decreasing scale and limited amount of discrete values. E.g. Controls to increase/decrease brightness, contrast, intensity or volume.

Example Use

<div id="ui-slider"></div>

Multiselect

What it is

Multiselect is a control that allows you to choose from a list of items. This JQuery control adds the ability to sort that list and to filter from the possible options to help make it easier to find what is wanted from a long list.

What Problem Does This Solve?

It removes the need for seperate selection controls for ordering and selecting. It adds a more intuative drag and drop interface for selecting and ordering. The search box makes it easier to find the item that is desired from a long list of possible items.

When to use it

Use when selecting and ordering from a long list of posible options. Particularly useful for when choosing and ordering columns for JqGrid.

Example use

Markup

 <select id="countries" class="multiselect" multiple="multiple" name="countries[]">
        <option value="AFG">Afghanistan</option>
        <option value="ALB">Albania</option>
        <option value="DZA">Algeria</option>
        <option value="AND">Andorra</option>
        <option value="ARG">Argentina</option>
        <option value="ARM">Armenia</option>
        <option value="AUT" selected="selected">Austria</option>
        <option value="FRA" class="mandatory" selected="selected">France</option>
        <option value="GEO">Georgia</option>
        <option value="DEU" selected="selected">Germany</option>
        <option value="ISL">Iceland</option>
        <option value="IND">India</option>
        <option value="IDN">Indonesia</option>
</select>

Best practices

  • Choose sensible defaults for selected items
  • Avoid mandatory fields unless necessary
  • If it is a short list consider checkboxes for selection
