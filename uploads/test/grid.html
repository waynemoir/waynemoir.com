Percentage based

The fluid grid system uses percents for column widths instead of fixed pixels. It is based on the column ratios from 960gs to allow for correct proportions for all resolutions and devices.

Markup

<div class="row">
        <div class="span4">4</div>
        <div class="span8">8</div>
</div>

Basic layout

As shown here, a basic layout can be created with two "columns", each spanning a number of the 12 foundational columns we defined as part of our grid system.

Nesting columns

To nest your content, just add a new .row and set of .span* columns within an existing .span* column.

The number of columns is reset at each level so that each row takes up 100% of the parent column. For example, two nested .span6 columns could be placed within a .span4.

Limiting page width Controlling the grid

To make the most of the users display, by default the page width uses the entire width of the browser.

As is the nature with these things there may be times in which the design demands the page width be limited. Achieving this is as simple as adding the .constrained class to the body. This works by setting a max-width to ensure consistancy throughout the application.

Markup

<body class="constrained">

Heads up!

The max-width property is not supported in Internet Explorer 6, users of this browser will always recieve a full screen layout.
